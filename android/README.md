# Life OS — Android client

Нативное Kotlin + Jetpack Compose приложение. Никакого WebView — обычный
Android-клиент, который ходит в REST API сервера (`../server`).

## Локальная разработка

Нужен Android Studio (Koala/2024.1+) с установленным Android SDK — на этой
машине его пока нет, поэтому первая сборка проверялась через GitHub Actions,
а не локально. При первом открытии проекта в Android Studio:

1. `File → Open` → выбрать папку `android/`.
2. Дать Android Studio синхронизировать Gradle (использует встроенный Gradle
   первого раза — обёртки `gradlew` в репозитории намеренно нет, чтобы не
   тащить бинарный `gradle-wrapper.jar` вслепую; Android Studio сама её
   восстановит через `Tools → Gradle → Generate wrapper`, если понадобится
   собирать без IDE).
3. Запустить конфигурацию `app` на эмуляторе или физическом телефоне.

## Настройка адреса сервера

По умолчанию приложение обращается на `http://192.168.219.107:8000`
(адрес Raspberry Pi на момент разработки). Меняется в приложении:
вкладка «Настройки» → поле адреса сервера → «Сохранить». Значение хранится
в DataStore на телефоне.

Так как сервер пока работает по обычному HTTP в локальной сети,
`AndroidManifest.xml` включает `usesCleartextTraffic="true"`.

## Что уже есть

- «Сейчас»: текущий блок, обратный отсчёт до конца блока, кнопки
  «Завершить»/«Пропустить».
- «День»: список блоков на сегодня.
- «Настройки»: адрес сервера + ручная проверка обновлений.
- Автопроверка обновлений при каждом запуске (GitHub Releases), см.
  `app/src/main/java/com/lifeos/app/update/UpdateChecker.kt`.

## Чего пока нет (сознательно отложено)

- Экраны «Неделя» и «Проекты».
- Редактирование/добавление/удаление блоков (пока только в веб-версии).
- Пуш-уведомления о начале следующего блока.
