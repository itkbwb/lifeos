Передача проекта от ChatGPT к Claude
Версия проекта: v0.2.0
Дата передачи: 2026-07-22

1. Контекст

Пользователь хочет персональную систему управления жизнью и временем на ближайшие 10 недель, а затем долгосрочный Life OS.

Главная цель ближайшего периода:

1. Нанопровода, приоритет 100
2. Статья Сонъёля, приоритет 90
3. Статья InAs в протравленных дроплетами канавках, приоритет 80
4. Подготовка к Германии, приоритет 85
5. Построение научной инфраструктуры, приоритет 50
6. Домашний сервер, приоритет 30

Приложение должно не просто хранить список задач. Оно должно показывать пользователю, что делать прямо сейчас, сколько осталось до конца блока и что будет следующим.

Пользователь хочет режим с подъемом в 07:00, четырьмя приемами пищи, залом 6 раз в неделю, суббота без зала, работой в институте и поздним возвращением домой.

6. Важная логика

Файл app/seed.py автоматически:

1. создает проекты
2. создает утреннюю рутину
3. создает вечернюю рутину
4. заполняет расписание с текущего дня до 15 августа 2026 года
5. не создает блоки повторно, если на дату уже существует хотя бы один блок

Это временная логика MVP.

Не удалять и не пересоздавать пользовательскую базу без прямого согласия пользователя.

11. Технические требования

1. Не терять существующий data/lifeos.db
2. Перед миграцией базы делать резервную копию
3. Использовать миграции при изменении схемы
4. Не полагаться только на seed.py для реальных пользовательских данных
5. API должен возвращать понятные ошибки
6. Все даты хранить и обрабатывать с учетом часового пояса Asia/Seoul
7. Не использовать datetime.utcnow для долгосрочной логики без явной timezone awareness
8. Проверить переход через полночь
9. Проверить блоки сна, которые могут пересекать дату
10. Не предполагать, что end_time всегда в тот же календарный день
11. Не кэшировать API ответы service worker так, чтобы пользователь видел устаревшие данные
12. Интерфейс должен работать при ширине экрана от 320 px
13. Кнопки должны быть удобны для Samsung S24 Ultra
14. Код и комментарии в коде писать на английском
15. Объяснения пользователю писать на русском

12. Известные ограничения v0.2.0

1. Сон представлен коротким блоком до 23:59, а не полноценным блоком до 07:00 следующего дня
2. Нет timezone aware backend
3. Нет авторизации
4. Нет CSRF защиты
5. Нет миграций
6. Нет задач внутри блоков
7. Нет повторяющихся шаблонов как самостоятельных объектов
8. Нет полноценной offline синхронизации
9. При отсутствии сервера UI может загрузиться из кэша, но изменения не ставятся в очередь
10. seed.py создает расписание на основании наличия любого блока на дату
11. проектная статистика показывает только текущую неделю
12. процент выполнения дня основан на количестве блоков, а не на длительности
13. фактическое время не записывается
14. невозможно вручную начать будущий блок
15. невозможно продлить текущий блок
16. невозможно перенести оставшийся день одним действием
17. service worker обновление нужно отдельно протестировать
18. PWA установка требует HTTPS или localhost
